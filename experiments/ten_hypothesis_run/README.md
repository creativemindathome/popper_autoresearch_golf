# Multi-hypothesis experiment runner

End-to-end runs that generate architecture hypotheses with **Anthropic Claude**, run the **falsifier** (Stage 1 gates T2–T7, optional Stage 2 adversarial prosecution), and record **knowledge graph** snapshots plus visualization data for timelines and clips.

## Prerequisites

- **API key:** `ANTHROPIC_API_KEY` (set in the environment or project-root `.env`).
- **Repo root on `PYTHONPATH`:** scripts add the repo root automatically; run them from this directory or via paths below.
- **Optional:** `ffmpeg` (or `moviepy`) for assembling MP4s; see `visualization/`.

## Canonical entry points

| Entry | Purpose |
|--------|---------|
| [`run_full_live_experiment.py`](./run_full_live_experiment.py) | **Main runner.** Creates `live_run_YYYYMMDD_HHMMSS/` with logs, `summary.json`, per-run JSON, graph snapshots, and `visualization/visualization_data.json`. |
| [`run_full_experiment.sh`](./run_full_experiment.sh) | Shell wrapper: runs the live experiment (default `--num-hypotheses 10`), then optional frames + video. Forwards extra CLI args to the Python runner (e.g. `--disable-stage2`). |

### Quick start

```bash
cd experiments/ten_hypothesis_run
set -a && source ../../.env && set +a   # if you use .env

python3 run_full_live_experiment.py --num-hypotheses 10
# or
bash run_full_experiment.sh
```

### Common CLI flags (`run_full_live_experiment.py`)

- `--num-hypotheses N` — how many hypotheses to generate (default: 10).
- `--output-dir PATH` — fixed output directory instead of auto `live_run_*`.
- `--disable-reviewer` — skip the novelty reviewer gate.
- `--disable-stage2` — Stage 1 only (faster).
- Reviewer/stage-2 model overrides: `--reviewer-model`, `--stage2-model`, etc. (`--help` for full list).

## Output layout (`live_run_*`)

```
live_run_YYYYMMDD_HHMMSS/
├── logs/
│   └── run.log
├── output/
│   └── run_NNN.json
├── graph_snapshots/          # optional, when enabled
├── visualization/
│   ├── visualization_data.json
│   └── frames/               # after generate_frames.py
├── summary.json
├── evolution_timelapse.mp4     # after assemble_clip.py
└── viewer.html                 # when generated by the pipeline
```

Use the `latest` symlink after `run_full_experiment.sh` completes, or open the most recent `live_run_*` directory.

## Visualization

From a completed run directory:

```bash
python3 visualization/generate_frames.py \
  --viz-data live_run_.../visualization/visualization_data.json \
  --output live_run_.../visualization/frames \
  --max-frames 100

python3 visualization/assemble_clip.py \
  --frames live_run_.../visualization/frames \
  --output live_run_.../evolution_timelapse.mp4 \
  --method auto
```

[`visualize_all_live_runs_timeline.py`](./visualize_all_live_runs_timeline.py) aggregates multiple `live_run_*` folders for a combined timeline (see script docstring).
